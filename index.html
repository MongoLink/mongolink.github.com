MongoLink by BodySplash

An Object Document Mapper (ODM) for Java and MongoDB

MongoLink was conceived to leave no print at all (or almost) on your project : no annotations, no super class to inherit. Mapping is declared through a fluent api directly in Java, and the configuration to get started is just a few lines long.

Dependencies

See the parent pom.xml

Install

For the time being, the simplest solution to use MongoLink is to clone the repository and build it with Maven. MongoLink also provides a set of tools to ease unit testing. Mainly, it provides an in memory implementation of MongoDB's drivers, as long with with a jUnit rule to setup your tests easily.
